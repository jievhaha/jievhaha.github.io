zookeeper常用命令

本文最后更新于：2021年2月28日 晚上

zookeeper常用命令

进入bin目录，执行如下启动命令客户端。

./zkCli.sh

ls命令

ls 命令用于查看某个路径下目录列表。后边必须跟路径

ls /

ls2（3.6版本已经不支持了，需要ls -s）命令用于查看某个路径下目录列表，它比 ls 命令列出更多的详细信息。

ls -s /

ls path [watch]可以对节点进行监听，但是不推荐使用了，推荐ls -w path替代。

注意：监听效果就一次，比如我接着delete /test，那会直接删除，不会再有下边WATCHER::之类的话了，还想监听，那就重新设置一次。

get命令

get 命令用于获取节点数据和状态信息。

可选参数[watch]对节点进行事件监听。已经过期，可以使用get [-w] path代替，和ls一样。

get /
get -s /

stat命令

stat 命令用于查看节点状态信息。

stat /

create命令

create 命令用于创建节点并赋值。

create [-s] [-e] path data acl
[-s] [-e]：-s 和 -e 都是可选的，-s 代表顺序节点， -e 代表临时节点，注意其中 -s 和 -e 可以同时使用的，并且临时节点不能再创建子节点。
path：指定要创建节点的路径，比如 /runoob。
data：要在此节点存储的数据。
act：访问权限相关，默认是 world，相当于全世界都能访问。

set命令

set 命令用于修改节点存储的数据。

set path value [version]

delete命令

delete 命令用于删除某节点。

delete path [version]
java zookeeper
